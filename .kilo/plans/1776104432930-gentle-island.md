# Plan: Report Analysis Feature

## Goal
Add an "Analyze Report" button to the burnout assessment UI that triggers a behavioral analysis using ground truths provided in the system prompt.

## Ground Truth (to be embedded in prompt)
> **Blink Rate (BR):** The blink-rate varies with emotional/physical stimulus. When captivated/interested, blink rate slows and declines as interest piquies. Increasing/rapid blink rate indicates high-stress, low concentration. Rapid blinking during conversation can indicate superiority and contempt.

## Analysis Parameters
Only analyze these fields from each survey entry:
- Question ID
- Domain
- Score (1-5)
- Voice Sentiment (positive/neutral/negative)
- Blink Rate Change (%)
- BR Stress Level (derived from blink rate change)

## Implementation Plan

### 1. Frontend: Add "Analyze Report" Button
**File:** `app/frontend/src/components/ui/detailed-report.tsx`
- Add new "Analyze Report" button next to Close button
- Button triggers analysis function with report data
- Show loading state during analysis
- Display analysis results in a new panel/modal

### 2. Frontend: Create Analysis Results Component
**New File:** `app/frontend/src/components/ui/report-analysis.tsx`
- Display analysis output in readable format
- Show insights with evidence (rule + data point)
- Highlight correlations, contradictions, patterns
- Include confidence indicators

### 3. Frontend: Add Analysis State
**File:** `app/frontend/src/App.tsx`
- Add state for `analysisResult` and `isAnalyzing`
- Pass to DetailedReport component

### 4. Backend: Analysis Endpoint/Tool
**Option A - HTTP Endpoint:** Create endpoint `/api/analyze-report` in `app/backend/app.py`
- Accepts report data (biometric snapshots)
- Returns JSON analysis using ground truth rules

**Option B - WebSocket Tool:** Add tool in `rtmt.py`
- Register new tool `analyze_report`
- Returns analysis as tool response

**Recommended:** Option A (HTTP endpoint) - simpler for button-triggered requests

### 5. System Prompt for Analysis
```
You are a behavioral analysis engine.

You MUST follow these rules strictly:

GROUNDING RULES:
- Use ONLY the provided "Research Rules" and "Input Data"
- Do NOT use external knowledge, assumptions, or general psychology
- If a conclusion cannot be derived from the rules, return "insufficient_evidence"

RESEARCH RULES:
Blink Rate (BR): The blink-rate varies with emotional/physical stimulus. When captivated/interested, blink rate slows and declines as interest piquies. Increasing/rapid blink rate indicates high-stress, low concentration. Rapid blinking during conversation can indicate superiority and contempt.

EVIDENCE REQUIREMENT:
- Every insight MUST include: the exact rule used, the exact data point used

OUTPUT RULES:
- Output MUST be valid JSON
- No explanations outside JSON
- No additional commentary

ANALYSIS RULES:
- Identify correlations between score and biometric change
- Highlight contradictions (e.g., high score + stress signal)
- Detect repeated patterns across questions
- Be conservative: prefer "insufficient_evidence" over guessing

CONFIDENCE:
- High: clear rule match + strong signal
- Medium: partial match
- Low: weak or borderline signal
```

## Acceptance Criteria
1. "Analyze Report" button appears on detailed report view
2. Clicking button sends data to backend with ground truth prompt
3. Results display in consultative tone with evidence
4. JSON output follows specified format with confidence levels
5. Graceful handling when insufficient evidence exists

## Notes
- The ground truth will be embedded in the backend prompt, not sent from frontend
- Frontend sends only the report data (biometric snapshots)
- Keep analysis lightweight - avoid heavy LLM calls if possible
