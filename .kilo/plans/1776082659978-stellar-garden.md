# Plan: Detailed Survey Report with Biometric Data Capture

## Goal
At the end of the burnout survey, display a detailed report capturing biometric changes at each question answer.

## Required Report Format

| Column | Description |
|--------|-------------|
| Question ID | Identifier (q1-q5) |
| Domain | Domain of the question (e.g., Emotional Exhaustion) |
| Score | Likert score (1-5) inferred from answer |
| Voice Sentiment | Positive/Neutral/Negative |
| Blink Rate Change | Delta from baseline (percentage change from baseline) |
| BR Stress Level | Stress level from blink rate analysis |
| Face Emotion | Dominant emotion detected |

---

## Implementation Plan

### Phase 1: Data Capture Infrastructure

**1.1 - Extend survey response tool (rtmt.py)**
- Modify `record_survey_response` tool to accept biometric parameters
- Add fields: sentiment, blink_rate_change_percent, face_emotion

**1.2 - Frontend biometric data flow**
- Capture `blink_rate_change_percent` from useBiometrics.ts (already computed)
- Capture current face_emotion from useVideoCapture.ts
- Send all data together with survey response

### Phase 2: Backend Tool Updates

**2.1 - Update `record_survey_response` tool signature**
```python
record_survey_response(
    question_id: str,      # q1-q5
    score: int,            # 1-5
    voice_sentiment: str,   # positive/neutral/negative
    blink_rate_change_percent: float,  # percentage change from baseline
    face_emotion: str      # HAPPY/SAD/ANGRY/etc.
)
```

**2.2 - Send biometrics to frontend**
- After recording response, send full biometric snapshot to frontend
- Frontend stores for report generation

### Phase 3: Frontend Report Display

**3.1 - Survey response state management**
- Extend survey state to store biometric record per question
- Store as array of BiometricSnapshot objects

**3.2 - New component: DetailedReport**
- Create `components/DetailedReport.tsx` component
- Render table with all columns
- Calculate totals and averages

**3.3 - Update App.tsx**
- After survey completion (5 questions), show detailed report view
- Display immediately after agent gives verbal summary

---

## Data Flow

### At each question answer:
1. Frontend captures: `blink_rate_change_percent` from biometrics hook
2. Frontend captures: current `face_emotion` from video capture
3. Backend sends `sentiment.update` after analyzing voice response
4. All data sent with `record_survey_response` tool call

### Existing data sources (already available):
- **Blink rate change %**: `useBiometrics.ts` → `blink_rate_change_percent`
- **Face emotion**: `useVideoCapture.ts` → current emotion from latest frame  
- **Voice sentiment**: Backend sends via `sentiment.update` message

## Implementation Steps

### Step 1: types.ts - Add BiometricSnapshot
```typescript
interface BiometricSnapshot {
  questionId: string;
  domain: string;
  score: number;
  voiceSentiment: 'positive' | 'neutral' | 'negative';
  blinkRateChange: number;  // percentage
  faceEmotion: string;
}
```

### Step 2: rtmt.py - Extend record_survey_response
- Add parameters for biometric data
- Store snapshot
- Send to frontend via tool response

### Step 3: App.tsx - State and display
- Store biometric snapshots in survey state
- After 5 questions, render DetailedReport

### Step 4: DetailedReport.tsx - Component
- Table with columns: Question ID, Domain, Score, Voice Sentiment, Blink Rate Change, Face Emotion
- Summary row with totals/averages
